# Contributing

Thank you for your interest in contributing to this project! This document provides guidelines and information for contributors.

## Ways to Contribute

- **Bug Reports**: Report bugs using the [Bug Report template](https://github.com/D-Heger/VintageStoryCalculator/issues/new?template=bug_report.yml)
- **Feature Requests**: Suggest new features using the [Feature Request template](https://github.com/D-Heger/VintageStoryCalculator/issues/new?template=feature_request.yml)
- **Code Contributions**: Submit pull requests with bug fixes, improvements, or new features
- **Documentation**: Help improve documentation, README, or code comments
- **Testing**: Test the application and report issues or suggest improvements

## Development Setup

### Prerequisites

- A modern web browser (Chrome, Firefox, Safari, Edge)
- Git for version control
- A text editor or IDE (VS Code recommended)

### Getting Started

1. **Fork the repository** on GitHub

2. **Clone your fork** locally:

   ```bash
   git clone https://github.com/YOUR_USERNAME/VintageStoryCalculator.git
   cd VintageStoryCalculator
   ```

3. **Create a feature branch**: (Or a fix/refactor/etc. branch as appropriate)

   ```bash
   git checkout -b feature/your-feature-name
   ```

4. **Install dependencies and start the dev server**:

   ```bash
   npm install
   npm run dev
   ```

5. **Make your changes** and test them

6. **Commit your changes**:

   ```bash
   git commit -m "feat(/refactor/etc.): brief description of your changes"
   ```

7. **Push to your fork**:

   ```bash
   git push origin feature/your-feature-name
   ```

8. **Create a Pull Request** on GitHub

## Code Guidelines

### General

- Follow the existing code style and patterns
- Write meaningful commit messages
- Update documentation as needed
- Ensure changes don't break existing functionality

## Testing

### Manual Testing

- Test all calculators with various inputs
- Verify calculations are accurate
- Test on multiple browsers and devices
- Check responsive design on different screen sizes

### Browser Compatibility

Test on:

- Chrome (latest)
- Firefox (latest)
- Safari (latest)
- Edge (latest)

## Pull Request Process

1. Ensure your code follows the guidelines above
2. Update the README.md and CHANGELOG.md as needed
3. Test your changes thoroughly
4. Write a clear PR description explaining:
   - What changes you made
   - Why you made them
   - How to test the changes
5. Link any related issues
6. Wait for review and address any feedback

## Issue Reporting

When reporting bugs or requesting features, please use the appropriate GitHub issue templates:

- [Bug Report](https://github.com/D-Heger/VintageStoryCalculator/issues/new?template=bug_report.yml)
- [Feature Request](https://github.com/D-Heger/VintageStoryCalculator/issues/new?template=feature_request.yml)
- [General Issue](https://github.com/D-Heger/VintageStoryCalculator/issues/new?template=general_issue.yml)

## Code of Conduct

Please be respectful and constructive in all interactions. This project follows a code of conduct to ensure a positive environment for all contributors.
